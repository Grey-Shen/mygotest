1.1 系统架构

系统架构

UDMP 主要由三个核心组件构成：

  1. 对象存储系统（PabOS）

    PabOS 是一个软件定义、云规模的分布式对象存储系统，用于存储非结构化数据文件。PabOS 具有高可靠、强安全、低成本、可扩展等特性，支持大批量高并发的数据读写，支持任意大小，任意格式的文件存储。

  2. 业务数据管理系统（UDMP Core）

    UDMP Core 是一个分布式架构的结构化数据管理系统，用于管理非结构化数据文件所附属的结构化业务数据，例如流水号、客户号、卡号、账号、票据号等。支持任意数量的业务数据字段，可以根据预先索引的业务数据字段快速查询结构化业务数据。

  3. 内容处理系统

    提供实时或异步地对非结构化数据进行计算处理的服务。例如鉴黄服务、视频转码服务、图片剪切缩放转格式服务、图片 OCR 识别服务、语音转文字服务等。内容处理系统同样也是分布式架构，处理能力可以动态扩展。

在三个核心组件基础上，UDMP 提供了一套更贴近我行业务层的 RESTful API 以及对应的 Java SDK 对其进行封装。业务系统仅需调用 UDMP SDK 设计简单的 Java 接口即可使用 UDMP 的所有功能。
