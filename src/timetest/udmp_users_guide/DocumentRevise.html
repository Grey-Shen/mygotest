3.6 文档修订

文档修订

用户可以在 Document 上传后，追加，替换或者删除关联的 Page 。

追加 Page 方法

用户 App 必须有目标文档的 Write 权限。

  1. 根据保存的 DocId 获取指定 Document。
  2. 使用获取的 Document 实例初始化新的 Page。
  3. 可以对新页面设置 Bizdata。
  4. 执行 Document#appendPage 方法保存追加的 Page 数据到 UDMP Core 和 PabOS 中。

代码示例

String docId;
Document doc = Document.find(docId);

FileInputStream stream = new FileInputStream(path);
Page page = doc.newPage(stream)
               .putIntoBizdata(key, value);

doc.appendPage("name of operator");

FileInputStream anotherStream = new FileInputStream(anotherPath);
Page anotherPage = doc.newPage(anotherStream)
                      .putIntoBizdata(key, value);

doc.appendPage("name of operator");

注意：

  1. 一次只能追加一个页面。如果需要追加多个页面，需要每次初始化新页面后，执行 Document#appendPage，再追加下一个页面。

替换 Page 方法

用户 App 必须有目标文档的 Write 权限。

  1. 根据保存的 DocId 获取指定 Document。
  2. 从获得的 Document 对象上，获取需要替换内容的 Page 实例。
  3. 执行 Page#replaceContentWith 方法保存替换的内容到 PabOS 中。

代码示例

String docId;
Document doc = Document.find(docId);
Page page = doc.getPage(pageNo);

FileInputStream stream = new FileInputStream(path);
page.replaceContentWith(stream, "name of operator");

注意：

  1. 替换后，原 Page 的下载地址不会改变。

删除 Page 方法

删除 操作实际是使用一个占位文件来替换原始文件。Page 本身不会从关联的 Document 中删除。

用户 App 必须有目标文档的 Write 权限。

  1. 根据保存的 DocId 获取指定 Document。
  2. 从获得的 Document 对象上，获取需要替换内容的 Page 实例。
  3. 执行 Page#destroy 方法从 PabOS 中“删除”对应的 Page 内容。

代码示例

String docId;
Document doc = Document.find(docId);
Page page = doc.getPage(pageNo);

page.destroy("name of operator");

注意：

  1. 删除后，原 Document 关联的 Page 数量不会变化。所有关联的 Page 对应的 PageNo 均不会变化。
  2. 删除后，原 Page 的下载地址依然可以访问，只是获取到的内容为占位文件的内容。
  3. 删除后，原 Page 的实例对象会处于过期状态。调用该对象的方法会抛出异常，以避免错误的使用。
