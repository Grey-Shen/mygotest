3.1 初始化 SDK

初始化 SDK

使用 UDMP SDK 的业务系统必须在启动阶段对 UDMP SDK 进行初始化，通过提交自己的 AppId 和 AppKey 向 UDMP 证明身份，同时获取底层的 PabOS 等配置信息。初始化过程必须通过 JSON 格式配置文件完成，以下范例中假设业务系统存储配置文件的位置为 /app/config 目录。

业务系统需要准备一个配置文件 udmp.json，内容如下：

{
  "webapi_endpoint": "http://api.udmpuat1.sdb.com.cn",
  "app_id": "YOUR_APP_ID",
  "app_key": "YOUR_APP_KEY",
  "log4j_config": "/app/config/udmp_log4j.xml"
}

注意：范例中的 webapi_endpoint 取值为测试环境 URL，在 UDMP 生产环境中应该是 http://api.udmp.pabos.com.cn。

然后使用以下 Java 代码初始化 UDMP SDK：

String udmpConfigFile = "/app/config/udmp.json";
UdmpConfig.init(udmpConfigFile);

注意：UdmpConfig.init() 方法只能被调用一次，建议在应用程序启动阶段执行初始化操作。
