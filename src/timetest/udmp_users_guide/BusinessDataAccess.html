1.4 业务数据访问方式

业务数据访问方式

将业务数据以文档形式组织并保存到 UDMP 之后，业务系统必须在自己的业务数据库中保存这些文档的 DocId，可选保存每个文档的文档页数量。

需要访问 UDMP 上保存的业务数据时，业务系统仅需提供 DocId，即可通过 Java SDK 获取并实例化一个 Document 对象，并可进一步获取 Document 内部的 Page 对象。

常规方式

已知业务数据字段名称前提下，业务系统可以通过 Document 对象上的 Java 接口，直接访问对应的结构化业务数据。

业务系统可以通过 Page 对象上的 Java 接口，以 InputStream 的方式访问 Page 对象内非结构化数据文件的内容。

高级方式

业务系统可以通过 Document 或者 Page 对象上的 Java 接口，获取 Page 对象内非结构化数据文件在 PabOS 上的 URL。任何系统或者个人，都可以通过这些 URL 访问或下载 PabOS 上存储的非结构化数据文件。

但是这些 PabOS 的 URL 具有时效性，默认仅在 2 小时内有效。因此业务系统不应该长期存储 URL，应该在需要 URL 的时候使用 Document 或 Page 对象上的 Java 接口重新生成。
