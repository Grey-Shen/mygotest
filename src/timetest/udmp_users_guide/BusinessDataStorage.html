1.3 业务数据存储方式

业务数据存储方式

结构化数据

业务数据组织方式中提到的结构化业务数据字段，以及文档自身的元数据，均为结构化数据，这些数据将会被存储到 UDMP 的 MongoDB Sharding 集群组件上。

业务层面的一个文档（Document）对象，在实际存储时，其结构化数据部分正好被存储为一个 MongoDB 的 Document 对象。在 MongoDB 中，一个典型的 Document 数据结构例子如下：

{
    "_id": ObjectId
    "doc_id": String # 文档 ID，由 API 自动生成。
    "app_id": String # 应用 ID，在应用申请过程中获得。
    "title": String # 用于描述文档内容的标题。
    "bizdata": { # 应用自行命名的文档业务数据。
        "a_field": "value",
        "another_field": "some value",
    }
    "pages": [ # 文档页内容，按页码顺序排列，要求必须存在，至少有一个元素
        {
            "key": String # 文档页在云存储上的 Key，要求必须存在
            "bizdata": { # 应用自行命名的文档页业务数据。
                "a_field": "value",
                "another_field": "some value",
            }
            "page_no": int32 # 文档页页码，必须大于 0
        },
        # 更多文档页……
    ]
}

非结构化数据

业务数据组织方式中提到的非结构化业务数据文件将会被存储到 UDMP 的 PabOS 组件上。
